# Contributing

RetailMind uses small, reviewable changes with tests at the boundary they affect.

1. Create a feature branch from `main`.
2. Copy `.env.example` to `.env`; never commit secrets.
3. Install the Node and Python dependencies described in the README.
4. Run `npm run verify` before opening a pull request.
5. Explain the user-visible change, test evidence, and any fallback behavior in the PR description.

Business rules belong in the provider-neutral Python domain layer. Catalog facts, prices, stock,
customer memories, and policies must not be generated by a language model. New provider adapters
must implement the existing protocol and preserve a deterministic fallback.
